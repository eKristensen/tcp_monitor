# Code Review Findings

Generated by multi-agent review of the initial Rust rewrite. All findings resolved.

| # | Area | Finding | Resolution |
|---|------|---------|------------|
| 1 | `server.rs` | Probe drain loop held semaphore permit indefinitely | Drain timeout added; probe semaphore is now separate from heartbeat |
| 2 | `server.rs` | Heartbeat and probe shared one semaphore; 32 probe conns could block all heartbeats | Independent semaphores: 128 heartbeat / 32 probe |
| 3 | `metrics.rs` | Inbound peer names permanently allocate label memory; no removal API | `try_register_server_peer` caps unique inbound names at `MAX_UNIQUE_SERVER_PEERS = 512` |
| 4 | `protocol.rs` | Handshake timeout admitted connection as `peer="unknown"` session | `recv_handshake` returns `Err` on timeout; caller closes without creating a session |
| 5 | `client.rs` | Peer address/port change on reload was silently ignored | Full `PeerConfig` equality check; address change triggers task restart |
| 6 | `client.rs` | Delayed echo arriving after timeout reset `consecutive_missed` to 0 | Echo seq validated against `sent_seq`; mismatch counted as a miss |
| 7 | `server.rs` | `recv_timeout` fixed at startup, not updated on reload | `Arc<AtomicU64>` shared between config-watch task and accept loop |
| 8 | `main.rs` | SIGTERM dropped all tasks abruptly, producing TCP RST on every session | Cancel token + `tokio::join!` drain ensures graceful FIN on shutdown |
| 9 | `config.rs` | Zero values for `reconnect_delay`/`heartbeat_interval` caused spin loops | Minimum value of 1 enforced in validation |
| 10 | `client.rs` | `node.name` captured once at startup, stale after SIGHUP | Read from live config each reload cycle |
| 11 | `server.rs` | `BrokenPipe` on echo write mapped to `"local_error"` | `BrokenPipe` now maps to `"connection_reset"` (remote-initiated) |
| 12 | `client.rs` | `next_send` advanced before echo wait; timeout caused immediate retry | After timeout, `next_send` reset to `now + heartbeat_interval` |
| 13 | `server.rs` | Echo path re-serialized packet instead of reusing the encode path | `protocol::encode_packet` shared by sender and echo |
| 14 | `server.rs` | Multiple failed handshakes shared `peer="unknown"` label, corrupting gauges | Connections failing handshake are closed immediately, no session or label created |
| 15 | `client.rs` | `ClientConfig` changes (heartbeat_interval etc.) not applied to running sessions | `run_manager` tracks `last_client_cfg`; all peer tasks restarted when it changes |
